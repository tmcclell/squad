### Core Finding: Trust > Billing
**Billing reality:** ~92 min/month — negligible cost.  
**Perception reality:** Users see unfamiliar automation and assume hidden costs. **Trust > math.**

### The Surprise Factor
**Current state:** User runs `squad init` → 10-20 workflow runs appear in Actions tab → "Is Squad spamming my quota? Am I getting billed?"  
**User experience:** Trust deficit. Loss of control. Out of alignment with transparency values.

### CLI-First Competitive Advantage
- **Unique differentiator:** "Zero surprise automation" — Squad does what you ask, when you ask
- **Perception shift:** "Squad does things to my repo" → "Squad does what I ask"
- **Versus competitors:** Cursor, Aider, GitHub Copilot all run locally. Squad's GitHub-native strength becomes an advantage if positioned as "transparent automation."

### Proposed Tier Model (3-Tier Opt-In)
**Tier 1: Manual CLI (Default)**
```
squad init                    # No workflows
squad triage                  # User explicitly runs commands
```

**Tier 2: Semi-Automated (Opt-In)**
```
squad init --with-automation  # Key workflows only
  - sync-squad-labels, squad-triage, squad-heartbeat
```

**Tier 3: Full Automation (Enterprise)**
```
squad init --with-full-automation  # All workflows + cron enabled
```

**Why:** Developers want explicit control, not opaque automation.  
**What:** Squad puts *you* in control. CLI-first. Actions are optional.  
**How:** Zero background automation by default. Opt-in when you're ready.  
**Outcome:** Trust, transparency, user autonomy.
