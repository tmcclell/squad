## Scope

Implement `--sdk` flag for `squad init` command to support two initialization modes:
- **Default (`squad init`):** Markdown-only, no config file
- **SDK Mode (`squad init --sdk`):** Generate typed `squad.config.ts` with builder syntax

   - Added flag parsing: `const sdk = args.includes('--sdk');`
   - Help text updated with `--sdk` option

2. **`packages/squad-cli/src/cli/core/init.ts`** (~114)
   - Options passthrough: `configFormat: options.sdk ? 'sdk' : 'markdown'`
   - Routing to generator functions based on format

3. **`packages/squad-sdk/src/config/init.ts`** (~337)
   - New function: `generateSDKBuilderConfig()`
   - Generates `squad.config.ts` with `defineSquad()`, `defineTeam()`, `defineAgent()` builder syntax
   - Config file skipped entirely when `configFormat === 'markdown'`

### Key Decisions

- **Markdown is default:** New users get zero config complexity by default (old Squad behavior)
- **Opt-in SDK:** Teams adopt builders explicitly when ready
- **Backward compatible:** Existing `configFormat: 'typescript'` and `'json'` remain supported (just not CLI-exposed)
- **Builder syntax:** Uses `@bradygaster/squad-sdk` builders for type safety
