### Strategic Recommendations
1. **Core Principle:** Squad should be CLI-first, user-controlled tool — not automatic bot service
2. **Migration Path:** 
   - v0.8.22: Add deprecation warnings + implement CLI commands
   - v0.9.0: Remove 5 workflows entirely
   - Post-v0.9.0: Offer opt-in automation for users who want it
3. **Zero Actions Required Vision:** Squad works with only 3 standard workflows (CI, release, docs). All Squad logic moves to CLI.
4. **UX Principle:** Users invoke Squad when they want it — no background automation without explicit opt-in.

### CLI Command Specifications
- `squad labels sync` — Sync labels from team.md
- `squad triage [<issue>]` — Triage issue(s)
- `squad assign <issue> <member>` — Assign to squad member
- `squad watch` — Watch for new work (polling)
- `squad labels check [<issue>]` — Check label consistency
